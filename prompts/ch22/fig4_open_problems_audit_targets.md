**Caption**: \textbf{Open problems as audit targets.} Each open problem pairs a system frontier with a diagnostic standard.

## Prompt

```text
Draw a wide research-frontier map for a graduate-level machine learning textbook.
Use the course's Zoom-inspired blue-white visual system.
LANDSCAPE orientation, wide and spacious.

CONCEPT:
Show seven open problems for model systems as audit targets. Each problem should connect a frontier question to a concrete sign of progress.

MAIN COMPOSITION:
Create a hub-and-spoke map. In the center, place a large audit lens labeled "MODEL-SYSTEM AUDIT". Around it, arrange seven problem cards in a clean oval. Each card contains:
1. the open problem name,
2. a short frontier question,
3. a "progress looks like..." micro-chip.

SEVEN PROBLEM CARDS:
1. COMPONENT ATTRIBUTION
Question: "which component caused the gain?"
Progress chip: "ablations + oracle tests explain improvements"

2. COST-RELIABILITY FRONTIER
Question: "is higher success worth the cost?"
Progress chip: "cost per success and quality per dollar reported"

3. CONTEXT FAITHFULNESS
Question: "did the model use the right evidence?"
Progress chip: "claim-level support audits become routine"

4. TRACE RELIABILITY
Question: "did the loop recover or drift?"
Progress chip: "trace diagnostics localize failures"

5. SCAFFOLDING GENERALIZATION
Question: "does workflow transfer beyond benchmark format?"
Progress chip: "survives perturbations and fresh templates"

6. BENCHMARK DECAY
Question: "has the benchmark saturated, leaked, or been gamed?"
Progress chip: "hidden, fresh, adversarial rotating tasks"

7. TOOL SAFETY AND OVERSIGHT
Question: "can tools be used without unsafe side effects?"
Progress chip: "permissions, sandboxing, escalation policies"

BOTTOM STRIP:
Add a final statement:
"Open problems become research programs when each has a diagnostic that would change our mind."

STYLE:
- Background: #FAFCFF
- Primary blue: #2D8CFF
- Light border: #CFE3F7
- Pale fills: stepped blues #B7D9F2, #D9EDFB, #E8F4FD
- Orange accent: #FF9F43 — use only on the central audit lens
- Text: #1A1A2E for headers, #6B7280 for annotations
- Clean sans-serif typography
- No gradients, no dark background, no 3D, no glow

IMPORTANT:
- Include exactly seven problem cards.
- Each card must include a question and a progress chip.
- Keep the center audit lens visually dominant.
- Fill the full canvas without crowding.
- Use only one orange accent.
- No product logos, benchmark logos, robots, or decorative mascots.
- Keep labels readable at 50% PDF width.
```
